# YTools 安全与外联审计

> 审计日期：2026-07-19
> 审计范围：仓库中的 Swift 原生应用、ModuleKit 与构建脚本。

## 结论

原生目标中没有广告、遥测、心跳、自动更新、外链跳转、插件市场、HTTP/MCP Server、网页容器或运行时下载。应用没有引入 `WebKit`、网络框架或第三方 Swift 依赖；运行时能力来自 AppKit/SwiftUI、Spotlight、Dictionary Services、Quick Look、Security/Keychain 和 ServiceManagement。

源码文档中的 Alfred URL 仅用于对标参考，不会被编译成可点击入口或由运行时访问。

## 外联与动态代码检查

| 检查项 | 结果 | 说明 |

| `WKWebView` / WebKit | 未发现运行时代码 | 只在架构文档的“明确不包含”段落出现 |

| `Process` | 1 个固定白名单调用 | 仅 `/usr/bin/pmset` + 固定参数 `displaysleepnow`；执行前验证路径可执行 |

## 敏感数据

- 剪贴板历史：AES-GCM 增量存储；随机 256-bit 密钥放在登录钥匙串，清单、独立记录和缩略图文件权限均为 `0600`，目录为 `0700`。钥匙串或加密失败时不会降级写明文，也不会用空数据覆盖无法读取的历史。
- 图片剪贴板：默认关闭，开启后单项最大 5 MB；原图记录与 192 px 缩略图分别加密，界面列表不常驻完整图片。加密记录总量上限为 200 MB。
- 敏感过滤：先过滤 Concealed/Transient/AutoGenerated 类型、常见密码管理器和用户指定 bundle ID，再构造和持久化记录。
- Snippets 与最近文档：各自独立 AES-GCM 存储和钥匙串密钥；最近文档只记录由 YTools 打开的本地非 App 路径。三类存储共享 `KeychainKeyAccessor` 实现，但 service/account 和密钥互不复用。
- 使用学习：只保存结果标识与规范化查询的 SHA-256，不保存明文查询或路径；28 天裁剪，可从设置清除。
- 剪贴板主线程只读取 `NSPasteboard` 快照；图片转换、缩略图、哈希、密文加载和持久化均由 Actor 完成。密钥或密文不可用时保留旧文件，新记录只留在当前进程内存并明确提示。

## 系统权限与风险边界

- 基础功能不申请辅助功能、联系人、日历或 Full Disk Access。只有用户确认执行 `empty` 时，才按需请求向 Finder 发送固定清空事件的自动化权限；拒绝后其他功能不受影响。
- 登录启动使用 macOS `SMAppService.mainApp`；失败时恢复真实状态并显示错误。
- Spotlight 搜索范围可由用户限制；当前 Swift Package 尚不是启用 App Sandbox 的正式 Xcode target。
- 自动粘贴、全局选择动作、联系人和日历尚未实现，因此不会静默请求相关权限。
- 文件复制和移动由独立 Actor 执行；应用扫描与文件导航由可取消的搜索 Actor 执行，避免主线程文件系统阻塞。
- 结果展开速度只是本机 `UserDefaults` 中的窗口动画时长；不改变搜索内容、结果排序、文件访问、权限或网络边界，并尊重系统“减少动态效果”。
- 启动器位置只在本机 `UserDefaults` 保存归一化坐标、显示器可用分辨率和系统显示器 UUID，用于跨屏幕映射；不采集屏幕内容，也不通过网络传输。
- 内置与个人模块全部由 `SearchCoordinator` 异步执行并经过 `ModuleResultPolicy` 二次校验：无权限模块不能打开文件，网页 URL 永远不能进入动作分发器，系统控制和应用退出动作只授予明确注册的内置模块。策略同时限制结果字段、复制载荷、数量和分数。
- 能力声明不是进程内沙箱；网络隔离由“源码随应用编译 + 禁止 API 构建扫描 + 主程序无网络实现”共同保证。未来网络能力必须进入独立 XPC Service，而不是增加没有执行层的 host 字符串声明。

## 发布前仍需完成

1. 使用完整 Xcode 建立 App target，启用 Hardened Runtime，并根据最终搜索范围决定 Sandbox/bookmark 策略。
2. 使用 Developer ID Application 签名、公证并验证 Gatekeeper；当前脚本只生成本机 ad-hoc 签名包。
3. 在未锁定桌面执行 VoiceOver、多屏、输入法、Quick Look、快捷键冲突和登录启动实机回归。
4. 每次新增网络 API 前，必须先增加独立能力声明、域名白名单、超时/响应大小限制和可关闭设置；主程序继续保持无网络实现。
