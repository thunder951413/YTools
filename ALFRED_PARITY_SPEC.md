# YTools × Alfred 逐项对标规格

> 版本：2026-07-19
> 范围：macOS 原生个人效率工具
> 原则：对标功能、信息层级和键盘效率；不复制 Alfred 源码、品牌、图标或专有视觉资产。
> 官方核对：Alfred Features Overview、Clipboard History、File Search、Universal Actions、Snippets、System Commands 与 Advanced/Force Keyboard（2026-07-19）。

## 1. 状态与边界

状态标记：

- ✅ 已达到当前验收标准。
- 🟡 已有实现，但行为、设置或边界尚不完整。
- ⬜ 尚未实现，属于计划内。
- 🚫 根据“本机、安全、无外部平台”的目标明确排除。
- 🔒 YTools 采用比 Alfred 更保守的安全策略。

优先级：

- P0：每天使用的启动、搜索、动作、剪贴板和快捷键体验。
- P1：明显提高效率的文件缓冲、Snippets、系统命令、设置完善。
- P2：联系人、日历、音乐等按需授权的系统集成。
- X：明确排除，不进入实现队列。

产品边界：

- 不做插件/Workflow 市场、远程脚本安装、遥测、广告、账户、心跳和启动联网。
- 不追求像素级复制 Alfred；视觉要有相同的信息效率，但保持 YTools 自己的图标、颜色和细节。
- 内置与个人扩展统一使用编译进应用的异步 Swift `YToolsModule`；需要网络的自用工具将来必须放入受限 XPC Service。
- Terminal、Web Search、浏览器书签和第三方密码库默认排除，除非用户以后明确改变安全边界。

### UI 尺寸基线

后续视觉实现以这些基线验收，不要求与 Alfred 像素相同：

- 主窗口默认宽度：820–900pt；可配置。
- 搜索区高度：68–76pt。
- 舒适结果行：58–66pt；紧凑结果行：44–52pt。
- 文件预览区：主窗口宽度的 35%–45%。
- 外层圆角：12–16pt；结果选中圆角：8–10pt。
- 选中背景必须在浅色、深色、降低透明度三种情况下清晰。
- 热唤起目标：面板显示到可输入不超过 100ms；按键到本地结果更新不超过 50ms；Spotlight 首批结果目标不超过 300ms。

### 快捷键设计规则

1. 全局快捷键必须至少包含一个修饰键。
2. 注册冲突时保留上一个可用快捷键，不允许功能静默失效。
3. 只有启用的功能才注册对应全局快捷键。
4. 所有面板内部快捷键必须在底部帮助或动作面板中可发现。
5. 输入法候选、VoiceOver 和系统保留组合优先，不得强行截获。
6. 需要辅助功能权限的快捷键与普通 Carbon 全局快捷键分开说明。

## 11. 非功能验收标准

### 性能

- 常驻空闲内存和冷启动数据需要在完整 Xcode App 构建后建立基线。
- 热键到窗口可输入：目标 ≤100ms。
- 本地同步模块结果：目标 ≤50ms。
- Spotlight 首批结果：目标 ≤300ms；超时显示非阻塞状态。
- 剪贴板监听不得持续占用明显 CPU；空闲轮询平均 CPU 接近 0%。
- 输入每个字符不得同步扫描整个磁盘或 Applications 目录。

### 稳定性

- 任意模块失败不能关闭主面板。
- Spotlight、词典、钥匙串和剪贴板失败必须显示可理解状态，不崩溃。
- 快捷键冲突必须保留上一个有效配置。
- 文件或应用在结果展示后被删除，执行时应提示“项目已不存在”。

### 隐私与安全

- 主程序没有启动联网、遥测、心跳、广告和在线更新。
- 剪贴板持久化必须保持 AES-GCM；钥匙串失败不得写明文。
- 敏感剪贴板类型与忽略应用先过滤、后持久化。
- 排序学习只保存哈希与聚合数据；清除学习应可在设置中执行。
- 联系人、日历、辅助功能、自动化等权限按功能单独请求。
- 不提供任意 Shell、HTTP Server、MCP Server、未签名动态模块和网页插件容器。

### 可访问性

- VoiceOver 能朗读结果标题、类型、副标题、位置和主动作。
- 完全不使用鼠标也能完成搜索、动作、返回、设置和清理确认。
- 支持减少动态效果、增加对比度、减少透明度和系统字体缩放。
- 快捷键录制器必须能通过键盘聚焦，并明确朗读当前组合键。

## 12. 实现批次与验收顺序

### Batch A：核心交互闭环（P0）

1. 动态窗口高度和统一 DesignTokens。
2. 右箭头动作面板与类型化 `ActionRegistry`。
3. 文件导航返回、Shift Quick Look、上下文底部提示。
4. 查询词哈希绑定学习与清除学习。
5. 快捷键冲突回滚、功能禁用时注销快捷键。
6. 设置搜索、搜索范围和结果类型设置。

验收：只用键盘完成“唤起→搜索→预览→动作→返回→设置”，无不可发现操作。

### Batch B：剪贴板与文件效率（P0/P1）

1. 图片历史和加密附件。
2. 忽略应用、分段清理和暂停记录。
3. 文件缓冲与批量动作。
4. 最近文档。
5. Snippets 浏览、保存和占位符。

验收：覆盖 Alfred Clipboard/File Search 的主要日常路径，同时默认不申请辅助功能权限。

### Batch C：系统原生工具（P1/P2）

1. 系统命令白名单和危险操作确认。
2. 拼写、Large Type、高级离线计算。
3. 联系人、日历、提醒事项按需授权。
4. 可选自动粘贴和文本展开。

验收：每项权限都能单独拒绝，拒绝后其他功能不受影响。

### Batch D：个人模块开发体验（P2）

1. 原生 UI 模板。
2. 模块能力声明。
3. Xcode 示例与测试模板。
4. 必要时提供隔离网络 XPC Service。

验收：能新增一个“查单词/处理文本/调用白名单 API”的自用工具，不引入网页容器或通用脚本权限。

## 14. 当前结论

YTools 已具备可日用的原生核心闭环：应用/文件搜索与导航、类型化动作、Quick Look、文件缓冲、最近文档、计算器与离线换算、词典/拼写、Large Type、加密剪贴板、Snippets、快捷键和可搜索设置。

当前更准确的状态是：

- 产品边界内的 Batch A 与 Batch B 主路径已经实现，并额外完成一批高价值 P1 工具。
- Alfred 的网页搜索、市场、Workflow 运行时、云同步、通用 Shell 等外部平台能力明确不复制。
- 尚未完成的重点是联系人/日历等 P2 授权集成、可选自动粘贴、Snippets 分类，以及更完整的系统控制和无障碍实机验收。
- 当前机器只有 Command Line Tools，可完成 Debug/Release 编译和临时签名；完整 XCTest、Developer ID、公证与辅助功能/VoiceOver 验收需完整 Xcode 和未锁定桌面环境。

后续每完成一项，必须同时更新本文件的“当前表现、状态、验收结果”，并进行真实窗口和键盘回归测试。
